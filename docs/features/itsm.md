# IT Service Management (ITSM)

Enterprise IT service management for internal IT operations, managed by the Platform Service with Workflow Service integration.

| Aspect | Implementation |
|--------|---------------|
| Incident Management | IT incident logging, categorization, prioritization, assignment, and resolution tracking with SLA |
| Service Request Catalog | Self-service catalog for IT service requests (provisioning, access, software) with approval workflows |
| Problem Management | Root cause analysis, known error database, and proactive problem prevention workflows |
| Change Management | Change request workflows with risk assessment, approval matrices, and change calendars |
| Configuration Management | IT asset configuration item (CI) registry with relationship mapping and dependency tracking |
| Knowledge Base | IT-specific knowledge base integrated with the platform Knowledge Management module |
| SLA Management | IT service level agreements with response/resolution targets, escalation rules, and reporting |
| Asset Management | IT asset lifecycle management from procurement through disposal with depreciation integration |
| Service Portal | Self-service employee portal for IT requests, incident reporting, and status tracking |
| CMDB Integration | Configuration Management Database with auto-discovery integration for network, server, and application inventory |
| Release Management | Release planning, deployment scheduling, rollback procedures, and post-deployment verification |
| Dashboards | IT service dashboards with MTTR, ticket volume, SLA compliance, CSAT, and trend analysis |

*See [Enterprise Job Scheduler](job-scheduler.md) for scheduled jobs. See [Knowledge Management](knowledge-management.md) for knowledge base. See [Architecture Overview](../architecture/overview.md) for system context.*
